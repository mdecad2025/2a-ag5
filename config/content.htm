Onshape 的優勢

Onshape 最大的亮點在於「即時協同」。它允許多位使用者同時在同一零件或組合件上進行編輯，變更會立刻同步給所有成員，就像多人同時編輯 Google 文件一樣。這大幅提升團隊效率，也減少因版本衝突或傳檔錯誤導致的問題。

Onshape 完全基於雲端運行，不需要安裝軟體，也不受電腦系統限制。使用者只需開啟瀏覽器即可操作，所有檔案都自動儲存在雲端，隨時可存取最新版本。這讓遠距或跨國協作變得非常方便。

Onshape 內建「分支（Branch）」與「合併（Merge）」功能，可同時嘗試不同設計方向，並在完成後合併回主線，類似程式開發中的版本控制（Git）。這對設計團隊在討論多種方案時特別實用。

Onshape 提供即時評論與標註功能，設計師可以直接在模型上留下意見或討論標記，並透過權限設定控制誰能檢視、評論或編輯。整個協作過程集中在同一介面完成，簡潔且高效率。

Onshape 採用自動更新機制，每次開啟系統時即為最新版本，不需使用者手動安裝或升級。此外，系統更新不會影響現有專案的穩定性，減少維護成本。

The Sphere node specifies a sphere centered at (0,0,0) in the local coordinate system. The radius field specifies the radius of the sphere.

(Sphere 節點指定了一個以當地座標系 (0,0,0) 為中心的球體。

radius 欄位用來指定該球體的半徑)

The subdivision field controls the number of faces of the rendered sphere. If the ico field is TRUE and the subdivision field is 0, the sphere is rendered as an icosahedron with 20 faces. If the subdivision field is 1 (default value), then each face of the icosahedron is subdivided into 4 faces, resulting in a 80 faces icosahedron. With a subdivision field set to 2, 320 faces will be rendered, making the sphere very smooth. A maximum value of 5 (corresponding to 20480 faces) is allowed for this subdivision field to avoid a very long rendering process. If the ico field is FALSE, then the sphere is rendered as a UV sphere, and the subdivision value has to be in the range [3, 32]. In this case the subdivision field specifies the number of rings and segments in which the sphere is subdivided. If the subdivision field is set to 3, then the sphere will have 3 rings and 3 segments, making 9 faces. With a subdivision field set to 32, the faces are 1024. The ico field specifies whether the sphere is rendered as an icosahedron (TRUE) or a UV sphere (FALSE). By default Sphere nodes created in Webots are icosahedrons with subdivision value of 1, whereas sphere geometries imported from VRML are UV spheres with a subdivision value of 24.

subdivision 欄位控制球體在渲染時的面數。
若 ico 欄位為 TRUE 且 subdivision 設為 0，球體將以 二十面體 (icosahedron) 方式繪製，共有 20 個面。
當 subdivision 設為 1（預設值）時，二十面體的每個面會被細分為 4 個面，總共變成 80 個面。
若 subdivision 設為 2，則會產生 320 個面，使球體表面更加平滑。
為避免渲染時間過長，subdivision 的最大允許值為 5（對應 20,480 個面）。

若 ico 欄位為 FALSE，則球體會以 UV sphere 方式繪製，且 subdivision 的值必須介於 [3, 32] 之間。
在此情況下，subdivision 欄位決定球體的「環數 (rings)」與「分段數 (segments)」。
例如：

  • 當 subdivision = 3 時，球體有 3 個環與 3 個分段，共 9 個面。

  • 當 subdivision = 32 時，球體共有 1024 個面。

ico 欄位用來指定球體是以 二十面體 (TRUE) 還是 UV 球體 (FALSE) 方式繪製。
在 Webots 中，預設建立的 Sphere 節點是 二十面體 (ico = TRUE) 且 subdivision = 1；
而從 VRML 匯入的球體幾何則為 UV sphere，subdivision = 24。

這段話在解釋 Webots 裡的「Sphere 節點」如何控制球體的外觀細緻程度與繪製方式。

ico決定球體類型:TRUE為二十面體，FALSE為UV球體(經緯度分割縱環、橫環)

過程與心得

  • 專注於幾何對齊： 在 Webots 中，最費時也最關鍵的步驟是確保所有的 HingeJoint 或 RotationalMotor 的 anchor 和 axis 參數與連桿的 translation 參數完美對齊。

    • 體會： 任何微小的座標偏差都會導致連桿在模擬開始時立即彈開或扭曲。必須像您分析問題一樣，逐步推敲每個連桿的局部座標系與世界座標系的關係，確保 x, y, z 軸和關節中心點的誤差。

  • 選擇正確的節點： 繪圖機需要精確控制，所以通常使用 RotationalMotor 作為主動關節（例如：連接地面的兩端），並使用 HingeJoint 作為被動關節（例如：您影片中要連接的 link 1 和 link 2）。

  • 在剛開始用webots的時候，確實有許多分支節點不知道怎麼運用，連接一個底座加上桿子就要花上幾十分鐘，更別說要將他們牢固的連接在一起，不過經過反覆觀看操作影片跟gemini的交流後，運用的也越加快速，哪裡可以匯入零件的外觀、哪裡要加HingeJoint將零件之間牢牢固定，最後還用了python將零件上的馬達運轉，總結來說，製作 Webots 繪圖機的體會就是：從嚴謹的幾何建模開始，通過精確的運動學數學模型，最終在控制器程式中實現時間與空間的精準控制。

L1, L2, L3, L4, L5 = 0.20, 0.27, 0.27, 0.20, 0.20
Ex = L5  # 兩馬達間距

# ================= 1. 逆向運動學 IK =================
def ik(cx, cy):
    """
    輸入: 目標座標 (cx, cy)
    輸出: 馬達角度 (單位: 弧度 rad), 如果無解返回 None
    """
    try:
        # --- 左臂 (t1) 計算 ---
        L_AC_sq = cx**2 + cy**2
        L_AC = math.sqrt(L_AC_sq)
        
        # 檢查工作空間
        if L_AC > (L1 + L2) or L_AC < abs(L1 - L2) or L_AC == 0:
            return None, None
            
        alpha = math.atan2(cy, cx)
        
        # 餘弦定理求 beta1
        cos_beta1 = (L1**2 + L_AC_sq - L2**2) / (2 * L1 * L_AC)
        cos_beta1 = np.clip(cos_beta1, -1.0, 1.0) # 數值保護
        beta1 = math.acos(cos_beta1)
        
        sigma1 = 1 
        t1 = alpha + sigma1 * beta1
        
        # --- 右臂 (t2) 計算 ---
        dx, dy = cx - Ex, cy
        L_EC_sq = dx**2 + dy**2
        L_EC = math.sqrt(L_EC_sq)
        
        if L_EC > (L4 + L3) or L_EC < abs(L4 - L3) or L_EC == 0:
            return None, None
            
        gamma = math.atan2(dy, dx)
        
        cos_beta2 = (L4**2 + L_EC_sq - L3**2) / (2 * L4 * L_EC)
        cos_beta2 = np.clip(cos_beta2, -1.0, 1.0)
        beta2 = math.acos(cos_beta2)
        
        sigma2 = -1 
        t2 = gamma + sigma2 * beta2
        
        return t1, t2

    except Exception:
        return None, None

# ================= 2. 影像處理與路徑生成 =================
def load_apple_points(image_path, n_points_target=100):
    if cv2 is None:
        print("Error: OpenCV is not installed.")
        return []
        
    # 嘗試讀取圖片
    img = cv2.imread(image_path, cv2.IMREAD_GRAYSCALE)
    if img is None:
        print(f"Error: 無法讀取圖片 {image_path} (請檢查檔名或路徑)")
        return []

    # 二值化與尋找輪廓
    _, binary = cv2.threshold(img, 200, 255, cv2.THRESH_BINARY_INV)
    contours, _ = cv2.findContours(binary, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_NONE)
    
    if not contours:
        print("Error: 圖片中找不到明顯輪廓 (請確認是黑線白底)")
        return []
        
    main_contour = max(contours, key=cv2.contourArea)
    
    # 重新採樣 (Resampling)
    if len(main_contour) > n_points_target:
        indices = np.linspace(0, len(main_contour) - 1, n_points_target, dtype=int)
        sampled = main_contour[indices]
    else:
        sampled = main_contour

    pts = sampled[:, 0, :]
    
    # 座標轉換
    h, w = binary.shape
    pts[:, 1] = h - pts[:, 1] # 翻轉 Y 軸
    
    # 正規化 (Normalize)
    center = np.mean(pts, axis=0)
    pts_centered = pts - center
    scale = np.max(np.linalg.norm(pts_centered, axis=1))
    if scale == 0: scale = 1
    pts_normalized = pts_centered / scale
    
    # 移至目標位置
    pts_scaled = pts_normalized * TARGET_RADIUS + TARGET_CENTER
    return pts_scaled

def generate_circle_points(cx=0.1, cy=0.3, r=0.1, steps=100):
    points = []
    for i in range(steps):
        angle = 2 * math.pi * i / steps
        px = cx + r * math.cos(angle)
        py = cy + r * math.sin(angle)
        points.append([px, py])
    points.append(points[0]) # 閉合
    return np.array(points)

    robot = Supervisor()
    timestep = int(robot.getBasicTimeStep())
    
    print("=== 程式開始執行 ===")

    # --- 步驟 A: 載入路徑點 ---
    target_image = "dog.jpg"  # 您的圖片檔名
    
    path_points = []
    
    try:
        print(f"正在嘗試讀取圖片: {target_image}")
        # 這裡會使用最上面定義的 N_POINTS
        path_points = load_apple_points(target_image, n_points_target=N_POINTS)
        
        if len(path_points) > 0:
            print(f"成功載入圖片路徑，共 {len(path_points)} 個點。")
        else:
            raise ValueError("圖片讀取結果為空")
            
    except Exception as e:
        print(f"注意: 圖片讀取失敗 ({e})。")
        print(">>> 自動切換為：圓形測試路徑 <<<")
        path_points = generate_circle_points(cx=TARGET_CENTER[0], cy=TARGET_CENTER[1], r=TARGET_RADIUS)

    # --- 步驟 B: 畫紅線 (視覺化) ---
    try:
        root = robot.getRoot()
        children_field = root.getField("children")
        line_node_str = generate_indexed_line_set(path_points)
        children_field.importMFNodeFromString(-1, line_node_str)
        print("已在地板繪製紅色路徑線。")
    except Exception:
        print("錯誤: 無法畫線。請確認機器人的 'supervisor' 欄位已設為 TRUE。")

    # --- 步驟 C: 取得馬達 ---
    t1 = robot.getDevice("t1")
    t2 = robot.getDevice("t2")
    
    if t1 is None or t2 is None:
        print("嚴重錯誤: 找不到馬達 't1' 或 't2'。請檢查 Webots 場景樹中的命名。")
        return

    # 設定馬達模式
    t1.setPosition(float('inf')) 
    t2.setPosition(float('inf'))
    t1.setVelocity(0)
    t2.setVelocity(0)
    
    # 角度轉換與 Offset 準備
    deg_to_rad = math.pi / 180.0
    offset1_rad = T1_OFFSET_DEG * deg_to_rad
    offset2_rad = T2_OFFSET_DEG * deg_to_rad
    
    print("開始路徑追蹤...")
    
    # --- 步驟 D: 先移動到起點 ---
    if len(path_points) > 0:
        start_x, start_y = path_points[0]
        q1_start, q2_start = ik(start_x, start_y)
        
        if q1_start is not None:
            t1.setPosition(q1_start + offset1_rad)
            t2.setPosition(q2_start + offset2_rad)
            t1.setVelocity(2.0)
            t2.setVelocity(2.0)
            # 等待走到起點
            for _ in range(50):
                robot.step(timestep)

    # --- 步驟 E: 畫圖迴圈 ---
    loop_index = 0
    while robot.step(timestep) != -1:
        
        # 循環播放路徑
        current_pt_idx = loop_index % len(path_points)
        x, y = path_points[current_pt_idx]
        
        # 1. 計算 IK
        t1_rad, t2_rad = ik(x, y)
        
        # 2. 如果算得出來，就移動
        if t1_rad is not None and t2_rad is not None:
            # 加上 Offset
            target_pos1 = t1_rad + offset1_rad
            target_pos2 = t2_rad + offset2_rad
            
            t1.setPosition(target_pos1)
            t2.setPosition(target_pos2)
            
            # 設定繪圖速度
            t1.setVelocity(3.0) 
            t2.setVelocity(3.0)
        else:
            # 超出範圍時印出警告(偶爾發生沒關係)
            # print(f"略過不可達點: {x:.2f}, {y:.2f}")
            pass
            
        # 3. 簡單延遲 (Plotter 效果)
        # 數字越大畫越慢，越小越流暢
        wait_steps = 5 
        for _ in range(wait_steps):
            if robot.step(timestep) == -1:
                break
        
        loop_index += 1

if __name__ == "__main__":
    main()
